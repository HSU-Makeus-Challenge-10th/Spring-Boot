## 🚀 미션 진행 방식

Spring Boot 세팅과 주차별 미션은 **반드시 본인 개인 폴더에서만 진행**합니다.

예시: `Junho/`

### 💡 진행 순서

1. **Issue 생성** <br>
   주차 별로 진행할 미션에 대해 이슈를 생성합니다.

2. **Branch 생성**<br>
   생성한 Issue를 기준으로 개인 브랜치를 생성합니다.

3. **작업 진행**<br>
   **본인 폴더 내부에서만** 진행합니다.
   다른 스터디원의 폴더는 **절대** 수정하지 않습니다.

4. **Pull Request 생성**<br>
   작업이 끝나면 PR을 올립니다.
   PR 본문은 저장소에 설정된 PR 템플릿 양식에 맞춰 작성합니다.
   PR은 반드시 본인 브랜치를 대상으로 작성합니다.

### 💡 작업 범위 규칙

- ✅ 본인 폴더 내 미션 코드 작성
- ✅ 본인 폴더 기준 리팩토링, 테스트, 문서 보완
- ❌ 다른 사람의 개인 폴더 수정
- ❌ 공통 구조를 임의로 변경하는 작업
- ❌ 다른 사람의 브랜치로 pr 작성



## 🤙🏼 컨벤션

### 💡 개인 브랜치명

- ✅ [이름]/[주차]

  ex) Junho/Week5

### 💡커밋 메시지

- ✅ [커밋태그] : [메시지]

  ex) feat : 댓글 등록 기능 추가  
  ex) fix : 댓글 중복 등록 오류 수정

💫 **commit 태그**

| 태그 | 설명 |
| --- | --- |
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| refactor | 코드 리팩토링 |
| comment | 주석 추가(코드 변경 X) 혹은 오타 수정 |
| docs | README와 같은 문서 수정 |
| setting | 프로젝트 세팅 |
| merge | merge |
| test | 테스트 코드, 리팩토링 테스트 코드 추가 |
| rename | 파일, 폴더명 수정 혹은 이동 |
| remove | 파일을 삭제만 한 경우 |

### 💡PR 제목

- ✅ [이름][미션내용]

  ex) [Junho] Week5 미션
