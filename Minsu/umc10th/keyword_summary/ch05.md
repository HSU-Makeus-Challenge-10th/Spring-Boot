## 빌더 패턴 (Builder Pattern)
`.field().field().build()` 체이닝 방식으로 객체를 생성하는 패턴. `@Builder` (Lombok) 하나로 사용 가능하며, 필드가 많거나 선택적 파라미터가 있을 때 가독성이 크게 향상됨.

## record vs static class
`record`는 Java 16+의 불변 데이터 전달용 클래스로, 생성자·getter·equals 등을 자동 생성. `static class`는 외부 인스턴스 없이 사용 가능한 중첩 클래스로 로직 포함 시 적합. 단순 DTO라면 `record`, 비즈니스 로직이 필요하면 `static class`를 사용.

## 제네릭 (Generic)
타입 파라미터(`<T>`)로 클래스·메서드를 일반화해 재사용성을 높이는 기능. `ApiResponse<T>` 같은 공통 응답 래퍼 클래스 생성 시 자주 활용. 컴파일 시점에 타입을 검사해 `ClassCastException`을 방지.

## @RestControllerAdvice
`@ControllerAdvice` + `@ResponseBody`가 합쳐진 전역 예외 처리 어노테이션. `@ExceptionHandler`와 함께 사용하여 컨트롤러 전체 예외를 한 곳에서 일관된 JSON 형식으로 응답.

## Optional
`null` 대신 값의 존재 여부를 감싸는 컨테이너 클래스로 `NullPointerException` 방지. JPA의 `findById()` 반환 타입으로 자주 사용되며, `.orElseThrow()`, `.orElse()` 등으로 null 처리를 명시적으로 표현.
