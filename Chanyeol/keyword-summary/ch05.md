- 빌더패턴이란?

  복잡한 객체의 생성 과정과 표현 방법을 분리하여 디양한 구성의 인스턴스를 만드는 생성 패턴

    - 목적: 매개변수가 많거나 복잡한 객체를 생성할 때, 가독성을 높이고 불변성을 쉽게 확보하기 위함
    - 장점
        - Telescoping Constructor(긴 생성자 체인) 문제 해결
        - 매서드 체이닝(Fluent Interface)으로 직관적인 코드 작성 가능
        - 객체 생성 단계별 검증 가능
        - 최종 객체를 불변으로 만들기 쉬움
    - 단점
        - 코드량이 증가
        - 단순 객체 사용 시에는 불필요
- record vs static class
    - record: 불변 데이터를 저장하고 전달하기 위한 인스턴스 중심의 데이터 구조
        - 데이터만 가지고 있는 구조에 최적화
    - static: 인스턴스화하지 않고 유틸리티 매서드나 전역 설정을 제공하는 정적 멤버
        - 상태 없이 기능만 제공할 때 주로 사용
- 제네릭이란?
    - 클래스 내부에서 사용할 데이터 타입을 외부에서 지정하는 기법
    - 목적
        - 컴파일 타임 타입 안전 보장
        - ClassCastException 같은 런타임 오류 방지
        - 코드 재사용성 증가
    - 장점: 타입 안전, 코드 중복 감소
- @RestControllerAdvice이란?

  스프링 프레임워크에서 컨트롤러(`@RestController`) 전역에서 발생하는 예외를 한곳에서 처리하는 어노테이션으로, **`@ControllerAdvice`와 `@ResponseBody`**가 결합된 형태이다.

- Optional이란?

  null이 될 수 있는 객체를 감싸는 Wrapper 클래스로 NPE를 방지하고 명시적으로 빈 값 가능성을 표현

    - 찾는 값이 명확하지 않은 경우
    - 설정값 조회, 검색 결과 등
