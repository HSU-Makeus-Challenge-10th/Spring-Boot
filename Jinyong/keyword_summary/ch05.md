- 빌더패턴이란?

  빌터패턴이란, 복잡한 객체를 단계별로 생성할 수 있게 해주는 디자인 패턴이다.
  클래스에 포함된 필드(매개변수)가 많거나 선택적인 값이 많을 때 주로 사용

  예를 들어 **서브웨이 샌드위치 주문**과도 같다.(빵 굽기는 미디엄, 햄은 저거, 채소는 저거…)
  정해진 레시피(기본 생성자)대로 완성품을 무조건 받아야 하는 것이 아니라, 재료를 단계별로 선택(메서드 체이닝)해서 나만의 샌드위치를 조립하는 방식이라고 할 수 있다.
  (원하지 않는 재료는 쉽게 뺄 수 있음!)

  **장점:**

  어떤 필드에 어떤 값을 넣는지 명확하게 보여서 코드가 읽기 쉬워지고(가독성) 유연한 객체 생성이 가능해진다.

- record vs static class
    - `record`는 자바 14부터 도입된 기능으로, 불변 데이터를 전달하기 위한 객체(DTO)를 쉽게 만들 수 있는 특수 클래스이다.
    - `static class`는 클래스 내부에서 독립적으로 사용하기 위해 선언하는 정적 중첩 클래스이다.

  예를 들자면,

    - **record**는 이미 모든 재료와 양념이 완벽하게 준비되어 있는 **밀키트**
    - **static class**는 **직접 요리**하는 것과 같다. 같은 결과물을 만들려면(Getter, Setter, 생성자 등)까지 직접 다 타이핑해야 한다.

  둘의 핵심적인 차이점은 `record`를 사용하면 생성자, `getter`, `equals`, `hashCode`, `toString` 같은 보일러플레이트(반복) 코드를 컴파일러가 자동으로 만들어주어 코드가 매우 짧아진다. 기본적으로 불변 객체라는 특징이 있다.

- 제네릭이란?

  제네릭(Generic)이란 클래스나 메서드 내부에서 다룰 데이터의 타입(Type)을 미리 정하지 않고, 외부에서 객체를 생성할 때 지정하도록 하는 기법이다.

    - 라벨이 붙은 정리 상자를 생각

  **장점:** 컴파일 단계에서 강력하게 타입 체크를 할 수 있어 안전하며, 불필요한 타입 변환(Casting) 과정을 생략할 수 있습니다.

- @RestControllerAdvice이란?

  `@RestControllerAdvice`이란 스프링 부트에서 여러 `@RestController`에서 발생하는 예외(Exception)를 한 곳으로 모아서 전역적으로 처리해주는 어노테이션이다.

    - ***학교의 보건실***을 생각해보면 비유가 쉽다.

  각 교실(Controller)에서 다치거나 아픈 학생(예외)이 생길 때마다 담임 선생님이 직접 치료(`try-catch`)하면 수업 흐름이 끊긴다.

  그래서 아픈 학생들을 모두 보건실(`@RestControllerAdvice`)로 보내면, 양호 선생님이 매뉴얼대로 일관되게 치료(에러 응답)해주는 원리다.

  **장점:**

    1. 컨트롤러에는 핵심 로직만 남기고, 지저분한 예외 처리 코드를 분리할 수 있어 코드가 깔끔해진다.
    2. 프론트엔드에 항상 일관된 포맷의 에러 응답을 내려줄 수 있다.(+ 본인도 유지보수 편해짐)

- Optional이란?

  null이 될 수도 있는 객체를 감싸서 안전하게 다루게 해주는 래퍼(Wrapper) 클래스이다.

    - 내용물이 들어있는지 모르는 **'택배 상자'**를 생각하면 쉽다.

  빈 상자인 줄 모르고 무작정 내용물을 꺼내려다 에러(`NullPointerException`)가 터지는 것을 막기 위해, 무작정 열지 않고 내용물이 있는지 먼저 확인하거나 비어있을 때 다른 걸 쓰라고 지시할 수 있게 해주는 포장지 역할을 한다.

  **장점:**

    - 실행 중 유효하지 않은 `null` 객체 참조에 접근하거나 메서드를 호출할 때 발생하는 런타임 예외인
      `NullPointerException`(NPE)을 안전하게 방지할 수 있다.
    - 코드에 `if (객체 != null)` 같이 지저분하게 붙어야 했던 예외 처리 코드를 줄여주어 가독성이 훨씬 좋아진다.
